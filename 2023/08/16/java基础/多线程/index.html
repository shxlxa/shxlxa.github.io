一、进程和线程

进程

在操作系统中运行的一个应用程序。每个程序之间是独立的，每个内存运行在独立的且受保护的内存空间内。

线程

  • 1个进程想要执行任务，必须得有线程。每个线程至少得有一个线程。

  • 一个进程的所有任务都在线程中执行

三、线程的状态

可以通过Thread.getState()方法获得线程的状态，一共有六种状态

  • NEW: 新建，尚未启动
  • RUNNABLE: 可运行状态
  • BLOCKED:阻塞状态
  • WAITING: 等待状态，在等待另一个线程

通过Thread.sleep方法暂停当前线程，进入WAITING状态

在暂停期间，若调用interrupt方法中断线程，会抛出java.lang.InterruptedException异常

四、线程的方法

  • thread.join方法，等待线程thread执行完成后，当前线程再继续执行任务。可以传指定最长指定最长等待时间。
  • thread.isAlive方法：查看线程thread是否还活着

五、线程安全问题

多个线程访问同一个对象、变量、同一个文件等时，可能会引发数据错乱和数据安全。

可以使用线程同步技术来解决线程安全问题

1.同步语句

原理

  • 每个对象都有一个与它相关的内部锁
  • 每一个执行到同步语句的线程可以获得obj的内部锁，在执行完同步语句后释放该锁
  • 只要一个线程持有了内部锁，那么其它线程在同一时刻将无法获得此锁。当他们视图获取此锁时，将会进入BLOCKED状态

obj必须是同一个对象，才能起到同步作用

2.同步方法

用synchronized关键字修饰的方法。注意：synchronized不能修饰构造方法
