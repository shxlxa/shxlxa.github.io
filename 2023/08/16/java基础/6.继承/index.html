继承

发表于2023-08-16|更新于2024-03-04|java
|阅读量:

关键字extends

任何类都继承自java.lang.Object,一般称为基类

java不支持多继承

方法的重写（override）

方法签名，包括方法名和参数类型。

重写：子类的方法签名与父类的一样。也叫覆盖。

构造方法必须先调用父类的构造方法

构造器

  • 子类是不继承父类的构造器（构造函数）的，它只是显式（有参数）或者隐式（无参数）地调用。

  • 如果父类的构造器无参数,子类会隐式调用，有参数的话需要显式调用。

  • class默认会添加无参的构造函数，如果构造了有参的构造函数，则不会添加。

继承的特性

  • 子类拥有父类非 private 的属性、方法。
  • 子类可以拥有自己的属性和方法，即子类可以对父类进行扩展。
  • 子类可以用自己的方式实现父类的方法。（即重写）

注解（Annotation）

3个常见的注解

  • @ovverride :告诉编译器这是一个重写后的方法

  • @SuppressWarnings (“告警类别”)：让编译器不生成警告信息

​ 如：@SuppressWarnings("unused")

  • @Deprecated:表示这个内容已经过期，不推荐使用

访问权限

java中有4个访问权限，从高到低如下所示：

修饰符 class package subclass world
public YES YES YES YES
protected YES YES YES NO
无修饰符 YES YES NO NO
private YES NO NO NO
  • public：在任何地方都是可见的
  • protected：仅在自己的包中、自己的子类中可见
  • 无修饰符：仅在自己的包中可见
  • private：仅在自己的类中可见

注意：顶级类只能用public和无修饰符修饰
